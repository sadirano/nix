# v0.10.0 release checklist

Manual verification for promoting `v0.10.0-pre` to the official `v0.10.0`.
Everything here is Windows-only (the supported platform).

**Sandboxing:** steps marked 🧪 must run against a scratch store, never the real
one. In a fresh PowerShell:

```powershell
$env:NIX_HOME = "$env:TEMP\nix-rc"        # scratch store for this shell only
```

Steps touching the **real** `~/.nix` are marked ⚠️ — snapshot first:

```powershell
Copy-Item ~/.nix ~/.nix-pre-0.10-backup -Recurse
```

---

## 1. Install & upgrade path

- [ ] Download `nix-v0.10.0-pre-windows-amd64.zip` from the GitHub release;
      `nix.exe --version` prints `v0.10.0-pre` (baked from the tag, not `0.0.0`).
- [ ] ⚠️ Deploy over the daily install (`nix-build.cmd` or drop-in + `nix --sync`);
      existing aliases, groups, actions, and `[shortcuts]` all still resolve.
- [ ] Confirm no machine still depends on the removed onix auto-migration
      (anything upgrading straight from ≤0.8.x must go through 0.9.0 first —
      note this in the release notes).

## 2. `--init` / PATH persistence (breaking: no pwsh snippet, no `--skip-profile`)

- [ ] 🧪 In a scratch shell: `nix --init` completes; **verify `$PROFILE` is
      untouched** (compare timestamps/content before and after).
- [ ] 🧪 `~/.nix/bin` (scratch) is on the *user* PATH in the registry
      (`Get-ItemProperty HKCU:\Environment -Name Path`).
      **Cleanup when done testing** — the scratch entry persists in the real
      registry; remove exactly it (type-preserving, no `setx`):

      ```powershell
      $bin = "$env:TEMP\nix-rc\bin"
      $k = Get-Item HKCU:\Environment
      $raw = $k.GetValue('Path','','DoNotExpandEnvironmentNames')
      $new = ($raw -split ';' | Where-Object { $_ -ne $bin }) -join ';'
      Set-ItemProperty HKCU:\Environment -Name Path -Value $new -Type $k.GetValueKind('Path')
      ```
- [ ] 🧪 All wrapper exes exist after init: `o e s y p r sg ff nix`.
- [ ] 🧪 `~/.nix/AGENTS.md` is (re)generated by `--init`/`--sync`.
- [ ] 🧪 No `~/.nix/shell/` dir is created on Windows; with a pre-existing
      `shell/nix.ps1`, `--sync` **deletes it** and prints the removal note
      (the `$PROFILE` dot-source warning + the `function q { exit }` tip).
- [ ] 🧪 `nix --init --skip-profile` now errors as an unknown flag.
- [ ] 🧪 `nix --doctor` shows no `shell` row on Windows; `wrappers`/`PATH`
      rows unchanged.
- [ ] A brand-new terminal (not a refreshed one) picks up the PATH and `o`/`r` work.

## 3. `--resolve` is read-only (breaking: no directory creation)

- [ ] 🧪 Register an alias, `rmdir` its directory, then `nix <alias>`:
      errors cleanly, **does not re-create the directory**.
- [ ] 🧪 Same for a segmented form (`nix docs@<alias>`): no directory created.
- [ ] 🧪 Registering a *new* alias (`nix name path`) still auto-creates the dir
      (creation moved to registration, not resolution).

## 4. `--export` / `--import` portable backup

- [ ] 🧪 `nix --export backup.toml` writes a file containing aliases + groups.
- [ ] 🧪 Delete one alias, `nix --import backup.toml`: **merge** restores the
      missing alias without overwriting ones that changed since the export.
- [ ] 🧪 `nix --import --replace backup.toml`: store matches the export exactly.
- [ ] ⚠️ Export the real store once and stash the file with the `~/.nix` snapshot
      (doubles as the rollback artifact for this release).

## 5. `--which` reverse lookup

- [ ] `nix --which <alias-dir>` prints the alias.
- [ ] `nix --which <alias-dir>\deep\nested\file` resolves to the same alias.
- [ ] With nested roots (one alias inside another), the **deepest** alias wins.
- [ ] `nix --which C:\Windows` (outside every alias) errors, empty stdout.
- [ ] Bare `nix --which` uses the cwd.

## 6. Child-process environment

- [ ] `r <alias> pwsh -c 'echo $env:NIX_ALIAS $env:NIX_ALIAS_PATH'` shows the
      alias name and absolute path.
- [ ] Same variables visible inside a saved action (`r <alias> :<name>`) and a
      `.nix/scripts/` script.
- [ ] `r <alias> pwsh -c 'echo $env:FZF_DEFAULT_OPTS'` is **empty** — the
      picker options must not leak into user subshells.

## 7. `--doctor`

- [ ] `nix --doctor` full report: all sections green on the daily machine.
- [ ] `nix --doctor -q` shows only problems + summary (empty-ish when healthy).
- [ ] `nix --doctor --json | ConvertFrom-Json` parses; sections present.
- [ ] With Everything running: doctor's `es` probe passes **and** `ff <alias>` /
      `s <alias> <pat>` actually return results (probe now uses the picker's
      exact `/ad` switches — these must agree).
- [ ] Stop Everything: doctor flags `es` as broken instead of passing falsely.
- [ ] Doctor flags duplicate alias sections in a deliberately doctored
      🧪 scratch `aliases.toml`.

## 8. Groups

- [ ] `o +<group>` opens the member picker; `r +<group> git status` fans out.
- [ ] Interactive: `s +<group> <pat>` and `y +<group> <pat>` pickers show
      `alias\rel` rows; picked files open / land on the clipboard.
- [ ] `p +<group>` group form works (clipboard → picked member dir).
- [ ] 🧪 `nix <unregistered>+<group>` routes through the register picker
      instead of failing.
- [ ] 🧪 `nix +g+g` (self-referencing add) is rejected.
- [ ] `o +nope` / `o +bad+token` errors — must **not** fall through to the picker.
- [ ] 🧪 Alias/group names with TOML metacharacters (`[`, `"`, `=`, …) are
      rejected with the friendly message.

## 9. Group usage & prune

- [ ] Use `o +<group>` once; the `usage` file gains a `+<group>` key and member
      aliases are *not* individually bumped.
- [ ] ⚠️ Interactive: `nix --prune` — members of a recently-used group rank by
      inherited recency (review the ordering, then **Esc out — don't delete**).

## 10. `[shortcuts]` wrappers

- [ ] 🧪 Add a `[shortcuts]` rename (e.g. `g = "sg"`), run `nix --sync`:
      new wrapper exe appears and dispatches correctly via argv0.
- [ ] 🧪 Rename it again + `--sync`: the **old** wrapper exe is deleted.
- [ ] 🧪 An invalid shortcut name is rejected at sync time.
- [ ] `--sync` while a shell window is open (wrapper exes potentially in use)
      still completes — atomic replace, no half-written exes.

## 11. Sweep & pickers

- [ ] Interactive: `nix --sweep` on the real index streams rows immediately
      (no long buffering pause before fzf appears) — **Esc out, don't delete**.
- [ ] `sg <alias> <pat>` and `ff <alias>` still behave normally end-to-end.

## 12. Machine-wide default actions (`~/.nix/actions/_default.toml`)

- [ ] 🧪 Create `_default.toml` with a `[actions]` entry; `r <any-alias> :<name>`
      runs it and `r <alias> :` lists it.
- [ ] 🧪 A project-local or central per-alias action of the same name wins.
- [ ] 🧪 `nix _default <path>` is rejected ("reserved").
- [ ] 🧪 `nix --export` includes `[actions._default]`; re-import restores it.

## 13. Release hygiene

- [ ] Release CI green: unit tests, e2e (56 checks), baked-version gate.
- [ ] Edit the auto-generated release notes to lead with an **Upgrading**
      section covering the breaking changes:
      - The PowerShell snippet is retired: the first `--sync`/`--init` deletes
        `~/.nix/shell/nix.ps1` — remove any `$PROFILE` dot-source line; if you
        used `q`, add `function q { exit }` to `$PROFILE`. PowerShell alias
        tab-completion is gone (the commands themselves are unaffected).
      - `nix --init --skip-profile` is no longer accepted (was a no-op).
      - Upgrades from ≤0.8.x must go through 0.9.0 first (onix auto-migration
        was removed).
- [ ] `v0.10.0-pre` is marked **Pre-release** on GitHub.
- [ ] The stable Scoop bucket did **not** pick up the pre (Excavator must keep
      pointing at v0.9.0 until the official tag).
- [ ] `scoop update nix-nightly` still works (nightly channel unaffected).

## 13. Promote

Once every box above is checked:

- [ ] `git tag -a v0.10.0 -m "…" && git push origin v0.10.0` (same commit as
      the pre unless fixes landed).
- [ ] Verify the release publishes as **Latest** (not pre-release).
- [ ] Wait for Excavator to bump the stable bucket — do not hand-edit it.
- [ ] `scoop update nix` on the daily machine; `nix --version` → `v0.10.0`.
- [ ] Delete this file (or reset it for the next cycle) and update the ROADMAP.
